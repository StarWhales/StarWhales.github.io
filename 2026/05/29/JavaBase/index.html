24.abstract（抽象）、static（静态）、final（最终）和this

只有static和final可组合在一起，而abstract和static一般不能组合在一起。

static（无 final）：可以多次赋值，不报错

final（无 static）：只能赋值一次，二次赋值编译报错

static final：只能赋值一次，二次赋值编译报错（常量）

static：大家共用同一个变量 → 可以改！

final：只能赋值一次 → 不能改！

static final：大家共用同一个常量 → 永远不能改！

static实际上是扩大了作用域，使同一个类的不同对象用的是同一个变量

final没有扩大作用域，换一个对象还是可以再赋值一次

25.接口

接口 = 统一标准 + 强制规矩

让所有类都遵守同一个规则，方便统一调用。

接口只有两样东西：

  1. 常量（变量值永远不能改）
  2. 方法声明（只有名字，没有方法体，必须重写）

如果类实现了接口，必须重写接口里的所有方法，并且可以直接使用接口里的常量。

40.异常处理

import java.lang.*;所有java程序默认导入，不用自己写

try：把 “可能出错的代码” 包起来。
catch：出异常了，就跳到这里接住并处理。
finally：不管有没有异常，一定会执行（用来关文件、关连接）。
throw：手动抛出一个异常（自己造错）。
throws：写在方法声明上，告诉别人 “我这个方法可能抛异常，你要处理”。
