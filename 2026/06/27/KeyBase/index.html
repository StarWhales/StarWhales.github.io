先加密再签名的危害

偷取A的消息，用自己的名义转发给B

签名没有被加密保护，攻击者提前把自己的公钥发给你，然后替换签名，替换身份（而非冒充，因为把公钥给你的时候你就记录了这个公钥是谁的）

仿射密码：

加密：E(x)=(a⋅x + b) mod 26
解密：D(y)=a^−1⋅(y−b) mod 26
x：明文
y：密文
a：乘法密钥，与26互素
b：加法密钥，0~25的整数

分组密码与流密码

分组密码：将明文分成固定长度的组，用相同的算法和密钥序列对每组加密到固定长度的密文

流密码：又称序列密码，每次加密1比特或1字节

混淆与扩散

扩散是雪崩效应，混淆是无法通过分析手段推断

混淆：让密钥和密文之间的关系变得极其复杂，看不懂。

扩散：让明文和密文之间的关系变得极其复杂，牵一发而动全身。

DES加密(64bit,8不同S盒)

  1. 64位明文，IP置换，有一张固定的表
  2. 16轮迭代，F轮函数：
    • E扩展：Lᵢ和Rᵢ，把32位Rᵢ扩展成48位，即分为8组，邻组头尾相接
    • 与密钥异或
    • S盒压缩：对于8组6bit数据，前后2bit组成行，中间4bit组成列，在S盒查询，6->4，8组数据依次用8个S盒
    • P盒置换：16次用同一张表
    • 左半 Lᵢ = Rᵢ₋₁
      右半 Rᵢ = Lᵢ₋₁ ⊕ F(Rᵢ₋₁, Kᵢ)
  3. IP逆置换
  4. 密钥扩展：
    • 64位初始密钥分为8组，PC-1置换（去掉每一组最后一位校验位，然后打乱顺序分为左C右D各28位）
    • 左移生成共16个子密钥，第 1、2、9、16 轮移 1 位，其他轮移 2 位。

ECB

构造Y = 0FFFFFFFFFFFFFFF（Y = X），搭配对应S，让X∣∣S按 16 位分组后的分组序列与Y∣∣X完全一致。

AES加密(128bit,相同S盒)

基于SPN代换置换网络

AES 类型 主密钥长度 轮数 轮密钥数量 每个轮密钥大小
AES-128 128 位 10 11 个 128 位
AES-192 192 位 12 13 个 128 位
AES-256 256 位 14 15 个 128 位

  1. 初始变换：16字节矩阵与16字节子密钥异或，子密钥有 轮数+1 个

  2. 10轮循环：

    • 字节代换：比如第一个字节是19，那就用S盒第1行第9列数据代换
    • 行移位：16字节矩阵有4行，第 i 行左移 i-1 位
    • 列混合：左乘一个固定矩阵，不是传统矩阵乘法（最后一轮不做这一步）
    • 轮密钥加：与子密钥异或

  3. 密钥扩展：

    密钥编排算法是面向字的(一个字由4个字节，32比特)。每个轮密钥由4个字组成。扩展密钥共44个字，表示为w[0],…,w[43]

    当 i 不是 4 的倍数：W[i] = W[i - 4] ⊕ W[i - 1]

    当 i 是 4 的倍数：W[i] = W[i - 4] ⊕ T(W[i - 1])

公钥体制

RSA

公钥和私钥制作过程：

  • 选两个质数 p，q，公钥 e
  • 公钥和私钥里相同的数字N = p * q
  • 欧拉函数T = (p - 1) * (q - 1)
  • 公钥E = (e, N)：
    • 质数
    • 1 < e < T
    • e不是T的因子
  • 算私钥D = (d, N)
    • (d * e) % T = 1

加密解密过程：（假设公钥为(7, 33)，私钥为(3, 33)，加密的数据为15，即N = 33, e = 7, d = 3）

  • 加密：
    • 15的7次方为170859375
    • 170859375 % 33 = 27
  • 解密：
    • 27的3次方为19683
    • 19683 % 33 = 15

ElGamal密码体制

椭圆曲线密码体制（ECC）

  1. 把明文数字 m 转换成椭圆曲线上一个点 Pm（明文嵌入，把数字映射成曲线上坐标点）
  2. B 随便选一个临时随机整数 k（一次性使用，用完丢弃）
  3. 计算两个点，拼成密文对 Cm = (kG, Pm + kPA)
    • 第一个分量：C1 = kG
    • 第二个分量：C2 = Pm + k⋅PA
  4. B 把 (C1, C2) 发给 A

Hash函数

Hash函数是一个将任意长度的消息（message）映射成固定长度消息的函数。

哈希碰撞

生日攻击法：假设哈希输出总共有 2^n 种不同取值（n 为哈希比特长度），当你生成大约 2^n/2 条不同消息并计算哈希值时，出现哈希碰撞（两条不同消息哈希结果一模一样）的概率约 63%。

MD5

  1. 增加填充位：
    • 在原始消息二进制末尾，先强制补一个二进制 1
    • 继续往后补二进制 0，直到整体长度模 512 等于 448
    • 最少填充 1 位，最多填充 512 位
  2. 附加原始消息长度
    • 统计原始消息的二进制总位数，把这个数字转为64 位二进制数
    • 将这 64 位拼接在步骤 1 填充完成的消息最末尾
  3. 初始化 MD5 缓冲区
    • A = 0x67452301
    • B = 0xEFCDAB89
    • C = 0x98BADCFE
    • D = 0x10325476
  4. 分组循环压缩：
    • 我们现在手里是1 个 512 比特的分组
    • 把这 512 位平均分：512÷32=16 份，记为 M0~M15
    • 把当前全局的四个寄存器 A、B、C、D 复制保存一份，记为 CVi = (A0, B0, C0, D0)
    • A = A + F/G/H/I 运算结果 + 分组字 Mj + 常数 T[i] mod 2^32，依次使用F、G、H、I四种非线性函数运算，每一种16次
      +
    • A/B/C/D新 = A/B/C/D迭代完 + A0/B0/C0/D0 mod 2^32
    • 叠加后的新 ABCD，作为下一个 512bit 分组的初始寄存器值，进入下一组运算
    • 所有 512bit 的分组运算结束后拼接ABCD转 32 位十六进制字符串即为哈希值

SHA-224、SHA-256、SHA-384 和 SHA-512 并称 SHA-2

DSS，DSA vs RSA

DSS 是标准，DSA 是该标准规定使用的签名算法

对比项 RSA DSA
用途 既可签名、也可加密 只能签名，不能加密
数学难题 大整数分解 离散对数
同消息签名 结果固定 随机k，每次签名不同
速度 签名慢、验签快 签名快、验签慢
签名形式 单个数值 二元组 (r,s)

PGP

安全电子邮件加密系统

发送方：

  • 原始消息 MD5
  • 己方 RSA 私钥签名
  • 邮件原文 + 签名 一起进行 ZIP 压缩
  • 生成 IDEA 随机密钥加密邮件压缩包(对称加密)
  • 对方 RSA 公钥加密 IDEA 密钥
  • 把加密邮件||加密的 IDEA 密钥打包发送
  • base64编码

接收方：

  • base64解码
  • 己方 RSA 私钥解密后半部分得到 IDEA 密钥
  • 用 IDEA 密钥解密邮件得到压缩包
  • 解压得到邮件原文 + 签名
  • 用对方 RSA 公钥验证签名

Shamir门限秘密共享

(t,n) 门限方案：
把一个总秘密k，拆分成n份子秘密（影子密钥），分给n个人保管，满足两条规则：

  • 凑齐 ≥ t 份子秘密：就能完整还原出原始总秘密k;
  • 拿到 < t 份子秘密：无法算出、推导出原始秘密k。t叫做门限值。

Shamir (t, n)门限(t - 1 次函数，子秘密分给 n 个人，1 <= t <= n)

利用 t - 1 次多项式唯一确定性：

  • 一个 t − 1 次多项式，必须至少 t 个不同坐标点才能唯一确定；
  • 少于 t 个点，存在无穷多个多项式满足这些点，无法解出原始秘密；
  • 把原始秘密设为多项式常数项，拆分多项式上 n 个点作为子秘密分发；
    刚好实现 (t,n) 门限特性。
