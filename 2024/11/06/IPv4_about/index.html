NAT Network Address Translation 网络地址转换：

IPv4 —> IPv6 过渡技术
①隧道技术
②双栈技术

内网通 外网通 做NAT

①定义内外网接口
Router(config)#int f 0/0
Router(config-if)#ip nat outside – 定义该接口为外接口
Router(config-if)#exit
Router(config)#int f 0/1
Router(config-if)#ip nat inside – 定义该接口为内接口
②匹配感兴趣流量 上网流量
ACL Access Control List 访问控制列表
作用：进行流量的匹配
Router(config)#access-list 1 permit 192.168.10.0 0.0.0.255(后面是反掩码）
③做转换
Router(config)#ip nat inside source list 1 interface f 0/0 overload
将来自内部的，跟列表1相互匹配的源流量在外接口f0/0做地址转换，将私有地址转换成公有地址，实现
上网。overload代表多对一上网。
④写去往全世界的路由
Router(config)#ip route 0.0.0.0 0.0.0.0 200.100.1.1 — 去往任意目的地，交给运营商转发

Router#show ip nat translations – 查看地址转换表

Inside local 原地址

交换机 VLAN

如果说不对交换机进行任何配置，一台交换机默认情况下就是一个广播域

即”傻瓜交换机” — 只起到连接作用

Switch#show vlan brief

VLAN Name Status Ports

1 default active Fa0/1, Fa0/2, Fa0/3, Fa0/4

​ Fa0/5, Fa0/6, Fa0/7, Fa0/8

​ Fa0/9, Fa0/10, Fa0/11, Fa0/12

​ Fa0/13, Fa0/14, Fa0/15, Fa0/16

​ Fa0/17, Fa0/18, Fa0/19, Fa0/20

​ Fa0/21, Fa0/22, Fa0/23, Fa0/24

1002 fddi-default active

1003 token-ring-default active

1004 fddinet-default active

1005 trnet-default active

默认存在1 思科设备还默认存在1002~1005

1个VLAN = 1个广播域

默认情况下，不同VLAN的用户无法直接通信

VLAN的数量0-4095 4096个

0和4095无法使用 1默认存在 思科设备中1002-1005私有

VLAN如何创建

①在配置模式下创建

Switch(config)#vlan 2

Switch(config-vlan)#exit

②在VLAN数据库中创建

Switch#vlan database

Switch(vlan)#vlan 4

Switch(vlan)#exit

交换机端口模式

①Access模式 接入模式 应用在连接终端的端口上

特点：只允许一个VLAN的数据通过

Switch(config)#int f 0/3

Switch(config-if)#switchport mode access – 配置交换机端口模式为Access模式

Switch(config-if)#switchport access vlan 3 – 配置该端口只允许VLAN3通过

Switch(config-if)#exit

②Trunk模式 主干模式 应用在连接交换机和交换机之间 交换机和路由器之间的链路上

特点：可以允许多个VLAN通过

Switch(config)#int f 0/1

Switch(config-if)#switchport mode trunk – 配置端口模式为Trunk

Switch(config-if)#switchport trunk allowed vlan 2,3 – 允许VLAN2和VLAN3的数据从该端口通过

Switch(config-if)#exit

思科三层交换机配置Trunk端口

Switch(config)#int f 0/1

Switch(config-if)#switchport trunk encapsulation dot1q

Switch(config-if)#switchport mode trunk
