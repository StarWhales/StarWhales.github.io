—信息收集知识

状态码：

状态码 状态描述 含义与应用场景
200 OK 请求成功，服务器返回请求的资源，是最常见的成功状态码
201 Created 请求已成功处理并创建了新资源（常用于 POST 请求创建资源，如新建用户、新增文章）
301 Moved Permanently 资源已永久移动到新位置，客户端应使用新 URL 进行后续请求（如网站域名更换）
302 Found 资源临时移动到新位置，客户端应继续使用原 URL（如临时跳转、活动页面临时迁移）
304 Not Modified 资源未修改，客户端可使用本地缓存的版本（用于 HTTP 缓存机制，减少重复请求）
400 Bad Request 请求语法错误或参数无效（如参数格式错误、必填参数缺失），服务器无法理解
401 Unauthorized 请求需要身份验证（如登录态失效、未携带 Token），客户端未提供或验证失败
403 Forbidden 服务器拒绝请求，客户端已通过身份验证，但没有目标资源的访问权限
404 Not Found 请求的资源不存在或已被移除（如访问不存在的页面、调用已删除的接口）
405 Method Not Allowed 请求使用的方法不被服务器支持（如用 POST 访问仅支持 GET 的接口、用 PUT 访问仅支持 DELETE 的接口）
500 Internal Server Error 服务器内部发生错误（如代码逻辑异常、数据库连接失败），无法完成请求处理
502 Bad Gateway 网关或代理服务器从上游服务器（如后端业务服务器）收到无效响应
503 Service Unavailable 服务器暂时无法处理请求（通常是服务器维护、资源耗尽，一般为临时状态）

任意用户行为

  • 验证码可爆破
  • 敏感的验证码信息直接包含在API响应包中 —> 拦截响应
  • 验证码修改为 **空 **或 **null **绕过
  • 手机验证码和手机号未绑定认证关系：
    • 用自己手机号发送数据包
    • 拦截改密码的请求
    • 将号码修改为别人的
  • 多账号发送验证码：（一次请求可以携带多个目标手机号，后端将为这些手机号生成（或复用）相同的验证码并发送）
    • 拦截发送验证码的数据包
    • 修改数据包为如下形式发送，两个手机号同时收到相同的验证码
  • 猜测万能验证码

支付逻辑漏洞

  • 修改生成订单的请求包，可以改数量或金额（可改为负数）
  • 四舍五入漏洞：
    • 充值0.019
    • 实际支付0.01
    • 实际到账0.02
  • 整数溢出漏洞：当支付金额为2147483649时，支付金额就变成了2
  • 修改优惠券漏洞：
    • 找到生成订单的数据包
    • 找到代表优惠券的参数，如 coupon_id
    • 爆破参数id，寻找可用的优惠券
  • 修改运费漏洞：修改生成订单的请求包

越权漏洞

网站鉴权方式：cookie，请求头。不断删除请求头中的键值对来确定身份验证字段

  • 水平越权：
    • 抓取查看数据的数据包，修改对应的ID值查看其他用户信息
  • 垂直越权：
    • 记录管理员删除用户的数据包请求参数，切换为普通用户构造数据包
    • 修改请求方式（put，post，get）绕过鉴权，例如响应中包含 Request method 'GET' not supported
      • 将get请求发送到重放器
      • 右键改为 post
      • Content-Type: application/x-www-form-urlencoded 改为 Content-Type: application/json
      • 给请求体加上 {}
    • 请求头伪造：网站在登陆之后服务器会返回一个认证标识，伪造认证标识有两个结果：
      • 情况1：严格验证，我识别不了我直接不准你访问
      • 情况2：我不认识，但是我感觉你很牛逼，我就直接放行，你是最屌的，直接变成管理员
    • 未授权访问漏洞：js文件泄露的接口未进行身份验证
