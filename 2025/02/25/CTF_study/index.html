—include_once($file)

返回值：

  • 如果文件成功包含，**include_once** 将会返回 **1**。
  • 如果文件没有被找到或者在尝试包含文件过程中有错误，**include_once** 不会返回任何值，但会产生一个警告（除非 error_reporting 被设置成忽略警告）。

各类文件类型的输出情况：

include_once 的参数不是一个 PHP 文件，那么该文件的内容会被直接输出（或执行，如果有可执行的 PHP 代码）到输出流中。具体的行为取决于该文件的内容和类型。

  1. 纯文本文件（比如 .txt）: 文件的内容会被直接输出。
  2. HTML 文件（比如 .html 或 .htm）: HTML 代码会被直接输出，浏览器会按照 HTML 来渲染。
  3. XML 文件或其他标记语言文件: 文件的内容会被直接输出。
  4. 二进制文件（比如图片或者音频文件）: 这样做通常不是个好主意，因为二进制数据可能会被错误地解释为文本，导致输出乱码或产生不可预测的结果。
  5. 包含 PHP 代码的非 PHP 文件: 如果文件中混合有 PHP 代码（即使文件扩展名不是 **.php**），那么那部分 PHP 代码仍然会被执行。

—前端验证与后端验证

  • 前端验证：通过JavaScript在浏览器中对用户输入进行验证（例如检查输入是否为合法的IP地址）。
    • 如果输入不符合要求（例如包含非法字符或命令），JavaScript会阻止表单提交或显示错误提示。关闭JavaScript后，前端验证失效，你可以输入任意内容（包括命令注入的Payload）。
  • 后端验证：在服务器端对用户输入进行验证。
    • 即使绕过前端验证，服务器仍会检查输入是否合法。

hint:select * from 'admin' where password=md5($pass,true)

ffifdyop 是一个特殊的字符串，它在某些情况下与 MD5 哈希的二进制表示有关。

假设有一个 SQL 查询如下：

SELECT * FROM users WHERE password = MD5('$input', true);

如果用户输入 **ffifdyop**，查询会变成：

SELECT * FROM users WHERE password = 'or'6\xc9]\x99\xe9!r,\xf9\xedb\x1c;

由于 **'or'6 **在 SQL 中是一个真值表达式，查询可能会返回所有用户的数据，从而绕过密码验证。

对于：

PHP 的 == 运算符在比较字符串时，如果字符串以 0e 开头（科学计数法表示 0），PHP 会将其视为数字 0。例如：

  • md5('240610708') 的哈希值是 0e462097431906509019562988736854。
  • md5('QNKCDZO') 的哈希值是 0e830400451993494058024219903391。
  • md5('QLTHNDT') 的哈希值是 0e405967825401955372549139051580
  • md5('s878926199a') 的哈希值是 0e545993274517709034328855841020

当 PHP 比较这两个哈希值时：

'0e462097431906509019562988736854' == '0e830400451993494058024219903391'

由于两者都以 0e 开头，PHP 会将它们都视为 0，因此比较结果为 true。

set sql_mode=PIPES_AS_CONCAT;

  • 这是 MySQL 的一个配置命令，用于设置 SQL 模式。
  • PIPES_AS_CONCAT 是 MySQL 中的一个 SQL 模式选项，它的作用是将 || 运算符解释为字符串连接符（类似于其他数据库，如 PostgreSQL 或 SQLite），而不是逻辑 OR 运算符。
  • 在 MySQL 中，默认情况下 || 是逻辑 OR 运算符，而不是字符串连接符。通过设置 PIPES_AS_CONCAT，可以改变 || 的行为。
  • set sql_mode=PIPES_AS_CONCAT; 只会影响当前会话的 SQL 模式。如果重新连接数据库，SQL 模式会恢复为默认值。

—waf绕过

1.空格绕过：

  • __${IFS}__（水平制表符）可代替空格，其中{}确保${IFS}不会被当作变量
  • %09（水平制表符），URL 解码发生在正则匹配之前

2.过滤{}下的空格绕过：

__$IFS$9__可代替空格，其中$9 是一个未使用的变量，通常为空，在 $IFS$9 中，$9 的作用是确保 $IFS 被正确解析为一个空格，若直接使用 $IFS，某些情况下可能会被解析为变量名的一部分，而不是空格。

3.字母绕过：

(~%8C%86%8C%8B%9A%92)(~%93%8C)

  • 目标字符：s y s t e m

  1. 获取每个字符的 ASCII 码：

    • s → ASCII 码 0x73
    • y → ASCII 码 0x79
    • t → ASCII 码 0x74
    • e → ASCII 码 0x65
    • m → ASCII 码 0x6D

  2. 计算原始十六进制值（取反后的值）：（~为取反运算符）

    • 0x73 取反 → 0x8C（二进制 01110011 → 10001100）
    • 0x79 取反 → 0x86（二进制 01111001 → 10000110）
    • 0x74 取反 → 0x8B（二进制 01110100 → 10001011）
    • 0x65 取反 → 0x9A（二进制 01100101 → 10011010）
    • 0x6D 取反 → 0x92（二进制 01101101 → 10010010）

  3. 组合成十六进制字符串：

    `~"\x8C\x86\x8C\x8B\x9A\x92" → "system"`

URL 编码的作用

  • 原始代码中直接使用 ~"\x8C..." 可能被过滤规则检测到特殊符号（如 \x），因此需要将十六进制值转换为 URL 编码：
  • \x8C → %8C
  • \x86 → %86
  • \x8B → %8B
  • \x9A → %9A
  • \x92 → %92
  • \x93 → %93

最终构造的 Payload 为：

  • (~%8C%86%8C%8B%9A%92)(~%93%8C)对应system("ls")

至于为什么解码后不是("system")("ls")呢？

因为在 PHP 中，如果一个字符串变量后面紧跟 ()，PHP 会尝试将其作为 函数名 调用。
